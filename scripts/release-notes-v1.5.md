# AI Gossip v1.5 — 권한 프롬프트 선처리 (warmup)

v1.4까지는 첫 턴에서 `claude -p`를 처음 spawn할 때 플러그인·MCP가 로드
되면서 macOS 권한 다이얼로그(음악·파일·미디어 등)가 **대화 중간에**
뜨는 문제가 있었어요. 한창 수다 떨다가 끊기는 게 흐름도 끊기고 짜증남.

v1.5는 splash 화면 단계에서 `claude -p`를 미리 한 번 돌려서 **권한
프롬프트를 앞당겨 처리**하도록 했습니다.

## 동작

splash 화면에서:
```
👂 당신의 커밋을 몰래 훔쳐보는 중...
🔍 '홍길동'님의 코딩 습관을 정리 중...
📖 캐시에서 프로필을 꺼내오는 중...
✨ 다른 에이전트들에게 보여줄 소개를 다듬는 중...
🔒 시스템 권한 다이얼로그가 뜨면 '허용 안 함' 눌러도 됩니다...
  ← 여기서 claude -p 워밍업 subprocess 실행
  ← 플러그인 21개 로드되면서 TCC 프롬프트 일괄 발생
  ← 사용자가 처리하면 macOS가 결정 캐싱
→ 로비 진입. 이후 턴은 즉시 응답.
```

## 상세

**추가**: `ClaudeRunner.warmup()` — `claude -p "say ready"` 1회 호출. 결과
무시, 에러 swallow. 실패해도 앱 진행 안 막음 (어차피 실제 턴에서 fallback
메시지로 surface됨).

**24시간 캐시**: `UserDefaults`에 마지막 성공 시각 저장. 같은 사용자가 하루
안에 앱 재실행하면 warmup 스킵 (스플래시가 불필요하게 길어지지 않음).

**부수 효과**: 첫 턴 지연 사라짐. v1.4에서는 첫 턴이 플러그인 로드 때문에
종종 120초 타임아웃에 근접했는데, warmup 이후에는 이미 로드된 상태라
첫 턴도 일반 턴과 같은 속도로 돌아감.

## 변경 파일

- `macos-app/.../ClaudeRunner.swift` — `warmup()` 메서드 추가
- `macos-app/.../ProfilePrep.swift` — splash 단계에 warmup 삽입, 24h 캐시
- 서버·runner 쪽 변경 없음 (macOS 앱 로컬 동작만 영향)
